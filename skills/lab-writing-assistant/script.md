# lab-writing-assistant — Execution Script

## Step 1: Load context

Read:
- `LAB_CONFIG.json` → writing_style, citation_format
- `LAB_MEMORY.md` → writing voice, preferences, known tics
- `research-graph.jsonl` → for the target project:
  - Hypotheses (H1, H0, status)
  - Linked papers (title, key finding, DOI/Zotero key)
  - Experiment results (if any)
  - Existing draft nodes (to avoid duplication)

If `--draft` provided: read existing file as context for extension/revision.

---

## Step 2: Security pre-flight

Check project sensitivity level (from research-graph.jsonl project node).

If `sensitive` or `confidential`:
> "⚠️ This project is marked as {level}. The draft will be generated using your local LLM context. Confirm you want to proceed with sending content to the configured LLM API? (yes/no)"

If user confirms or project is `internal`/`public`: proceed.

---

## Step 3: Build LLM prompt by section type

### `introduction`
```
Write an academic introduction section for a paper about: {project description}

Context:
- Field: {fields}
- Target journal style: {journal or "general academic"}
- Writing style: {writing_style from config}
- Key gap in literature: {hypothesis rationale from graph}
- Papers to cite: {list of linked papers with Zotero keys}

Structure:
1. Opening hook — broad context (2-3 sentences)
2. Narrowing to specific gap (3-4 sentences, cite {papers})
3. Statement of the problem (2-3 sentences)
4. What this paper does (2-3 sentences)
5. Brief roadmap (1-2 sentences)

Use [CITE:zotero_key] as citation placeholders.
Write in active voice. Be direct. Avoid filler phrases.
Target length: 400-600 words.
```

### `abstract`
```
Write a structured abstract for this paper:
- Background: {project description + gap}
- Hypothesis: {H1 text}
- Methods: {experiment design from graph}
- Results: {results from graph, or "[RESULTS PENDING]" if not yet}
- Conclusion: {implied from hypothesis}

Format: {journal abstract format if --journal provided, else IMRaD}
Target: 250 words max.
Use [CITE:key] for any citations needed.
```

### `methods`
```
Write a methods section for:
- Study design: {experiment nodes from graph}
- Participants/samples: {n, criteria from graph}
- Measures/instruments: {methods nodes from graph}
- Analysis plan: {analysis nodes or lab-biostat results from graph}
- Software/tools: {from graph or infer from field}

Style: past tense, third person, precise and replicable.
Include subsections: Participants, Materials, Procedure, Data Analysis.
Use [CITE:key] for methods references.
```

### `discussion`
```
Write a discussion section for:
- Main finding: {results from graph}
- Hypothesis tested: {H1} — supported/not supported/mixed
- Comparison to prior work: {linked papers and their findings}
- Limitations: {flagged in experiment nodes, or generate plausible ones}
- Future directions: {from advisor session notes or generate}

Style: {writing_style}. Avoid overclaiming. Acknowledge contradicting papers.
Target: 600-800 words.
```

### `grant-aim`
```
Write Specific Aim {n} for an NIH-style grant proposal:
- Overall goal: {project description}
- This aim: {hypothesis for this aim}
- Approach: {methods}
- Innovation: {what's novel — from lit scout gap findings}
- Impact: {why this matters}

Format: 1 paragraph, ~150 words. End with a clear expected outcome.
Use strong, active language. Avoid hedging.
```

### `response-to-reviewers`
```
Draft a response to reviewers for: {project paper}
Reviewer comments: {provided by user}

For each comment:
1. Thank the reviewer (briefly, not sycophantically)
2. Summarize the concern
3. Draft a substantive response
4. Note what changes were made (or will be made) in the manuscript

Tone: professional, confident, not defensive.
```

---

## Step 4: Generate draft

Call LLM with constructed prompt. Stream output.

Post-process:
- Replace any [CITE:key] placeholders with actual Zotero citation keys if available
- Flag [RESULTS PENDING] sections clearly
- Add header comment: `<!-- Draft generated by lab-writing-assistant on {date}. Review and edit before use. -->`

---

## Step 5: Save to Obsidian

```bash
cat > "{vault}/Research/Projects/{project}/drafts/{type}-{date}.md" << 'EOF'
{draft content}
EOF
```

---

## Step 6: Update research graph

Add Draft node to project:
```jsonl
{"type":"Draft","id":"draft_{hash}","project":"proj_X","section":"{type}","path":"{obsidian path}","status":"first-draft","created":"<ISO>","word_count":450}
```

---

## Step 7: Output to user

Print the draft inline in chat, then:
```
✍️ Draft saved to: Research/Projects/{project}/drafts/{type}-{date}.md

⚠️ This is a first draft. Review carefully before use.
Citation placeholders: [CITE:key] — replace with formatted citations.
Sections marked [RESULTS PENDING] need your data.

🏆 +200 XP | Badge: ✍️ Author
```

---

## Step 8: Award XP + log session

+200 XP for first draft of each section type per project. Log to sessions/.
