# lab-init — Execution Script

## Overview
Interactive onboarding. Ask the user questions conversationally, then generate all config and scaffold files. If files already exist, offer to update specific sections rather than overwrite.

---

## Step 0: Check existing setup

```bash
ls ~/.openclaw/workspace/lab/LAB_CONFIG.json 2>/dev/null && echo "EXISTS" || echo "NEW"
```

- If **NEW**: full onboarding flow below
- If **EXISTS**: ask "Want to (1) add a new project, (2) update preferences, or (3) full reset?"

---

## Step 1: Ask onboarding questions

Ask these conversationally — don't dump them all at once. Group naturally.

### Identity
- "What's your name?" (for LAB_MEMORY.md personalization)
- "What's your primary research field(s)?" (e.g. neuroscience, ML, climate science — can be multiple)
- "What disciplines does your work span?" (e.g. biomedical, computational, social science)
- "What career stage are you at?" (PhD student / postdoc / faculty / independent)

### Tools
- "Do you use Obsidian? If yes, what's the path to your vault?" (default: ~/obsidian-vault)
- "Do you use Notion? If yes, do you have a research database ID?"
- "Do you use Zotero? Local or web? If web, what's your library ID?"
- "Which databases do you mainly search?" (PubMed / OpenAlex / arXiv / Semantic Scholar — can pick multiple, default all)

### Preferences
- "How would you describe your writing style?" (e.g. concise and technical / narrative and accessible)
- "Preferred citation format?" (APA / APA7 / Vancouver / Nature / Chicago — default APA)
- "How do you prefer paper summaries?" (bullet points / short paragraph / detailed breakdown)
- "How many papers at a time is comfortable to review?" (default 5)

### Notifications
- "Where should I send weekly digests and alerts?" (discord / slack / none — default discord)

### First project (optional)
- "Want to set up your first research project now?" (yes/no)
  - If yes: ask project name, one-line description, main hypothesis (optional), status (idea/active/writing/submitted)

## Step 6: Create Obsidian folder structure (if vault provided)

```bash
mkdir -p "<vault>/Research/Projects"
mkdir -p "<vault>/Research/Literature"
mkdir -p "<vault>/Research/Methods"
mkdir -p "<vault>/Research/Weekly-Digests"
```

Create `<vault>/Research/README.md` explaining the folder structure.

If a first project was provided:
```bash
mkdir -p "<vault>/Research/Projects/<project-slug>"
touch "<vault>/Research/Projects/<project-slug>/hypotheses.md"
touch "<vault>/Research/Projects/<project-slug>/notes.md"
touch "<vault>/Research/Projects/<project-slug>/drafts/"
```

---

## Step 7: Register field-trend cron (if notify channel set)

Tell the user:
> "I'll set up your weekly field trend digest to run every Monday at 8am. You can change the time in LAB_CONFIG.json."

Register cron via OpenClaw:
```
openclaw cron add --name "lab-field-trend-weekly" --schedule "0 8 * * 1" --skill lab-field-trend
```

(Log this as a Yellow Line action in daily memory.)

---

## Step 8: Security baseline

Run a quick baseline:
- Check if any API keys are stored in plaintext in LAB_CONFIG.json → move to env or warn
- Note the Obsidian vault path is local — confirm it's not synced to a public repo

## Error handling

- Obsidian vault not found → warn, skip folder creation, note in LAB_MEMORY.md as "vault path unverified"
- Zotero not found → skip, note in config as null, suggest setting up later
- Cron registration fails → warn user to set up manually, provide the cron string
